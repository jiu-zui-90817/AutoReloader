# YR/MO Hot Reloader Toolchain (红警2/心灵终结 热重载工具链)

一套专为 **红警2：尤里的复仇 (YR)** 及 **心灵终结 (Mental Omega)** 开发的**前后端分离式动态热重载工具链**。

本工具链由两部分组成：
1. **后端注入引擎 (AutoReloader.dll)**：基于 `YRpp` 开发的独立注入组件，采用非侵入式 DLL 注入架构，可在游戏运行中实时监测并向内存写入修改后的 INI 属性。
2. **前端可视化控制台 (IDE)**：基于 Python 构建的图形化编辑器。内置数据回滚、状态校验及冗余代码自动清理功能，提供安全高效的可视化调试环境。

本项目针对心灵终结 (MO) 的调试需求进行了深度适配，支持各类复杂的视觉动画与特效（AE系统）的实时预览，显著提升 Mod 开发与调试效率。

---

## 🚀 核心特性

- **开箱即用**：提供预编译组件与独立运行的前端工具，配合专用启动器即可快速部署热重载环境。
- **全要素图纸覆盖**：支持游戏内核心实体与特效代码的动态覆写：
  - **作战单位 (TechnoTypes)**：载具、步兵、建筑、战机。
  - **武器火控系统**：武器 (Weapons)、弹头 (Warheads)、抛射体 (Bullets)。
  - **MO 特效系统 (AE)**：动画 (Anims)、粒子 (Particles)、粒子系统 (ParticleSystems)、Voxel动画 (VoxelAnims)。
- **状态同步与数据回滚**：前端 IDE 引入“基准数据重置”与“异步无感清理”机制。用户清空参数并保存时，工具将自动从原版底包提取默认值以重置内存状态，并在重载完成后自动清除 INI 文件中的冗余代码。
- **双向数据监控**：前端提供“所见即所得”的代码沙盒，后端提供可热插拔的调试控制台，实现配置文件与内存状态的精准映射。

## 📥 快速上手指引

### 1. 部署后端注入引擎
1. 前往 [Releases](../../releases) 页面，下载最新的整合压缩包。
2. 将解压出的 `AutoReloader.dll`、`ReloaderConfig.ini` 以及 `MOInjector.exe` 放入游戏根目录。
3. **必须使用 `MOInjector.exe` 启动游戏**。组件成功挂载后，将根据配置文件自动唤出调试终端。

### 2. 启动前端可视化控制台
1. 确保 `Codex_ZH.json` 字典文件存在或已通过数据提取脚本生成。
2. 运行前端主程序（`.py` 源码或打包后的 `.exe` 可执行文件）。
3. 建议将前端工具与游戏原版 `rulesmo.ini` 置于同一目录，以激活完整的“数据自动回滚”与“代码自动清理”功能。

## 💻 标准调试工作流

为避免传统修改器常见的“内存状态不同步（脏数据）”问题，建议采用以下标准流程：

1. **修改参数**：在前端左侧可视化表单中调整属性，或在右侧代码沙盒中手动编写进阶参数。
2. **保存部署**：按下 `Ctrl + S` 保存，代码将被写入 `hotfix.ini`，同时前端后台启动 5 秒的清理倒计时。
3. **触发重载**：立即切入游戏，按下热重载快捷键（默认 `F5`）（启用自动重载无需此操作），后端引擎将读取 `hotfix.ini` 并更新内存。
4. **自动清理**：5 秒倒计时结束后，前端会自动对比原版底层数据，将不再需要的冗余参数静默删除，保持补丁文件精简。
5. **恢复默认**：如需恢复原版属性，在前端清空对应参数并保存（或点击“恢复原版”），系统将自动下发默认值重置内存，随后清理残留代码。

## ⚙️ 配置文件说明 (`ReloaderConfig.ini`)

在游戏根目录下修改 `ReloaderConfig.ini` 可动态调整重载器行为（保存即生效，无需重启游戏）：

```ini
[Settings]
; 自动监控开关。true: 文件保存后立即触发内存更新；false: 需手动按热键触发
AutoMonitor=true

; 手动重载触发热键。默认 116 代表 F5 键 (采用 Windows 虚拟键码)
HotKey=116

; 调试控制台开关。true: 显示终端窗口以查看注入日志；false: 隐藏控制台
ShowConsole=true

; 目标监控文件列表。支持监控多个文件，使用英文逗号分隔。请确保包含前端部署的目标文件（如 hotfix.ini）
TargetINI=hotfix.ini, ae_effects.ini

```

## 🛠️ 编译指引 (面向 C++ 开发者)

如需基于后端注入器进行二次开发，请参考以下环境要求：

- **开发环境**: Visual Studio 2019 / 2022 (工具集 v142 / v143)
- **依赖库配置 (重要)**: 
  本仓库为了保持纯净，**未直接包含** `YRpp` 依赖库。在编译前，请务必：
  1. 前往 [YRpp 官方仓库](https://github.com/YRpp/YRpp) 下载最新源码。
  2. 将下载的源码解压，并放置于本项目的 `Backend/YRpp/` 目录下。
  3. 确保 Visual Studio 项目的“附加包含目录”能正确识别该路径。
- **编译参数**: 建议在项目属性中关闭“符合模式 (Conformance Mode: No)”，并指定 **C++17 标准**。
